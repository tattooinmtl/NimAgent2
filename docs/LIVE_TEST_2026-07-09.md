## What was fixed (and how each fix was verified)

| # | Bug | Fix | Verified by |
|---|-----|-----|-------------|
| 1 | Native-tools path leaked raw `<think>` blocks to the terminal (reasoning-only responses) and stored them in conversation history on every response | `stripThink`/`extractThink` now applies on **all** provider paths, not just template/text-tools ([src/core/agent.mjs](../src/core/agent.mjs)) | Mock provider replay: reasoning-only response now shows only `▸ Thinking (~8 words)`, stored history contains zero `<think>` tags |
| 2 | `saveSettings` persisted `.env`-supplied API keys into `settings.json`, and a crash mid-write could truncate the file to 0 bytes (observed — it holds API keys) | Env-injected keys tracked in runtime-only `_env` and restored to their on-disk value on save (unless deliberately changed via `/apikey`); write is now atomic (temp file + rename) ([src/core/config.mjs](../src/core/config.mjs)) | REPL run with `NIMAGENT_MOCKTEXT_KEY=env-secret-do-not-persist`: `/thinking on` saved settings, file kept `"not-needed"`, zero occurrences of the env value, file size intact |
| 3 | `ERR_USE_AFTER_CLOSE` crash when the REPL closed while a turn was still running (burst/piped input, stdin EOF) | `replClosed` flag guards `rl.prompt()` / `rl.pause()` / `rl.resume()` ([src/cli/repl.mjs](../src/cli/repl.mjs)) | Same burst-input scenario that crashed before the fix now exits cleanly (`bye 👋`, no stack trace) |
| 4 | `docs/` was fully gitignored, so `docs/EXTENDING.md` (referenced by the committed `/extend` skill) could never ship | `.gitignore` now uses `docs/*` with whitelists for `EXTENDING.md` and `TESTED_FEATURES.md` | `git check-ignore` no longer matches; both files appear as untracked |
